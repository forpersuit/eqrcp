# 一、先做结构调整（比视觉更重要）

### ❌ 当前问题

* 右侧信息块过重（QR + autosave + devices 堆叠）
* 左侧聊天区缺乏“内容主导权”
* autosave 不属于实时交互 → 不应该在主视图

### ✅ 调整原则

👉 聊天 = 主战场
👉 状态 = 辅助
👉 工具 = 悬浮 / 折叠

---

## ✔ 调整结果

### 左侧（主区）

```
[顶部]
连接状态 + 设备数 + 一键复制

[中间]
聊天流（核心）

[底部]
输入框 + 快捷操作
```

### 右侧（弱化为“控制面板”）

```
Card 1: Chat状态（极简）
Card 2: QR（可折叠）
Card 3: Devices
```

👉 **autosave 移除**
→ 放到：

* 设置页（推荐）
* 或输入框旁一个📁小入口

# 二、消息系统设计（重点）

你这个产品的本质不是聊天，是：

> **“传输可视化工具 + 轻聊天壳”**

所以消息要强调：
👉 信息类型
👉 传输状态
👉 操作效率

---

# 三、消息组件设计（逐个拆）

## 1️⃣ 文本消息

### 结构

```
[头像]  气泡
        文本内容
        时间 + 状态
```

### 设计要点

* 不要像微信 → 太重
* 用 **轻气泡 / 或无气泡（推荐）**

👉 推荐样式（更现代）：

```
你：
→ 右对齐
→ 绿色淡底 or 无底 + 边框

设备：
→ 左对齐
→ 灰色
```

---

## 2️⃣ 文件消息（重点）

### 结构（必须清晰）

```
[📄 icon] 文件名.ext
         2.3MB  ·  已接收
         [打开] [保存] [复制路径]
```

### 关键点

👉 文件不是消息，是“资源”

所以必须：

* 可操作按钮直接露出
* hover 才出现 → ❌ 错

---

## 3️⃣ 图片消息

### 结构

```
缩略图（最大宽度限制）
↓
文件信息（hover or 底部）
```

### 必须有：

* 点击放大（modal）
* 拖拽保存
* 右键复制

👉 不要加复杂边框 → 会很“土”

---

## 4️⃣ 视频消息

### 结构

```
[视频封面 + 播放按钮]
↓
时长 + 大小
```

### 关键点

* 不要自动播放
* 点击 → 内嵌播放器 or 弹窗

---

## 5️⃣ 传输中（最重要）

你这个产品核心价值就在这

### 必须有状态：

```
上传中…
[██████░░░░] 65%
1.2MB / 2MB
```

### 状态枚举：

| 状态        | 表现     |
| --------- | ------ |
| sending   | 进度条    |
| success   | ✔      |
| failed    | ❌ + 重试 |
| cancelled | ⏹      |

---

# 四、统一消息视觉语言

## 推荐风格（你这个产品最合适）

👉 “工具型 UI”（不是社交 UI）

### 核心原则：

* 去气泡化（弱气泡）
* 信息块化（像 Notion / Slack）
* 低对比
* 强可读

---

## 示例（你可以直接照着实现）

### 文本

```
你好，这是一段文本
12:30 ✓
```

### 文件

```
📄 report.pdf
2.3MB · 已接收

[打开] [复制路径]
```

### 图片

```
[ 图片缩略图 ]

1.2MB · 点击查看
```

# 五、交互升级（拉开档次的关键）

## 1️⃣ hover 操作层（高级感来源）

所有消息 hover 时出现：

```
复制 | 下载 | 删除
```

---

## 2️⃣ 右键菜单（必须有）

```
复制
保存
转发（未来）
删除
```

---

## 3️⃣ 拖拽支持（你这个产品必须做）

* 拖文件进来 → 发送
* 拖图片 → 发送
* 拖出去 → 保存

---

# 六、输入区（可以优化一大截）

## 当前问题

* 太“网页表单感”

## 改造：

```
[ + ]  输入内容...
       [🙂] [📎] [发送]
```

### + 按钮：

展开：

* 文件
* 图片
* 剪贴板

---

# 七、二维码区域（你现在的问题点）

## 当前问题

* 太“占空间”
* 信息密度低

## 改造：

👉 默认收起
👉 展开为卡片

```
Scan to join  ⌄
```

点击：

```
[ QR ]

192.168.xxx
[复制] [浏览器打开]
```

---

# 八、视觉风格建议（关键）

## 颜色

* 主色：Deep Teal（你之前设计已经对了）
* 背景：#F7F9FB
* 卡片：纯白
* 分割：极浅灰

---

## 圆角

* 卡片：12px
* 按钮：10px
* 输入框：20px（更现代）

---

## 阴影

👉 少用
👉 用“层级”代替阴影

---

# 九、总结（核心优化点）

你现在 UI 的问题不是“设计不好”，而是：

### ❌ 当前

* 功能堆叠
* 信息无层级
* 像后台工具

### ✅ 优化后

* 聊天流成为核心
* 传输状态清晰
* 操作路径最短
* 工具能力隐藏但随时可用

# Chat Bubble Attachment Layout Fix - 修复总结

## 问题描述

在聊天模式下，当消息包含附件（扁平图片或长文件名的文件）时，气泡容器在内部右侧留出不必要的空白区域。这导致气泡边框无法协调地包裹附件内容，影响视觉一致性和用户体验。

## 根本原因

1. `.message:has(.attachment-card)` 设置了固定的 `max-width: min(460px, 88%)`
2. `.attachment-card` 内部的附件有自己的宽度限制：
   - 图片附件：`width: min(300px, 100%)`
   - 文件附件：`width: min(280px, 100%)`
3. 当附件实际宽度小于消息容器的最大宽度时，`.bubble` 和 `.message-main` 会填充整个消息容器宽度，导致右侧出现空白

## 修复方案

在 `pages/pages.go` 文件中添加了两个CSS规则，使气泡和消息主容器能够自适应收缩以匹配附件的实际宽度：

### 修改1：消息主容器自适应宽度

```css
.message:has(.attachment-card) .message-main {
    width: fit-content;
}
```

**位置**：第936-938行
**作用**：让包含附件的消息主容器宽度自适应内容，而不是占满整个消息容器

### 修改2：气泡自适应宽度

```css
.message:has(.attachment-card) .bubble {
    background: #f8fbf5;
    padding: 6px;
    width: fit-content;
}
```

**位置**：第1096-1100行
**作用**：让包含附件的气泡宽度自适应内容，紧密包裹附件卡片

## 修复效果

### 修复前
- 气泡容器宽度固定为最大宽度（460px或88%）
- 当附件宽度小于容器宽度时，右侧出现空白区域
- 气泡边框不能紧密包裹附件内容

### 修复后
- 气泡容器宽度自适应附件实际宽度
- 气泡边框紧密包裹附件内容，无右侧空白
- 保持最大宽度限制，防止过宽

## 回归测试保证

修改使用了CSS的 `:has()` 伪类选择器，只影响包含 `.attachment-card` 的消息：

✅ **不受影响的场景**：
- 纯文本消息（无附件）
- 系统消息
- 正常比例的图片
- 视频附件
- 短文件名的文件附件

✅ **受益的场景**：
- 扁平图片（宽高比较大的图片）
- 长文件名的文件附件
- 任何实际宽度小于容器最大宽度的附件

## 技术细节

- **CSS属性**：`width: fit-content`
- **浏览器兼容性**：现代浏览器均支持（Chrome 46+, Firefox 94+, Safari 11+）
- **响应式设计**：保持了原有的最大宽度限制和响应式断点
- **性能影响**：无，仅为CSS样式调整

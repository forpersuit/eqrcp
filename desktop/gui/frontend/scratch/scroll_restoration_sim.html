🔬 EQT GUI 传输列表仿真诊断（100% 复刻）

由于设备在传输大文件时每 50ms 更新一次进度并覆盖 devicesWrapper.innerHTML：
  • 滚动冲突：每 50ms 重新创建 .devices-scroll-container 节点，导致其 scrollTop 滚动高度被浏览器自动清空（旧版表现）。
  • 点击吞噬（核心）：当用户点击设备行时，鼠标按下（mousedown）与鼠标抬起（mouseup）跨越了 DOM 的重构周期。因为被点击的物理 DOM 节点已经被销毁替换成了新节点，浏览器拒绝触发 click 事件，导致无法点击触发折叠展开（旧版表现）。
修复策略： 1. 使用 pointerdown (瞬时按下即响应，无需等待抬起) 接管折叠触发； 2. 使用局部 UI 更新函数直接就地刷新，并且保存与还原 scrollTop； 3. 在 agent-status 心跳高频更新时对局部重绘加入 250ms 节流以保证顺滑滚动。

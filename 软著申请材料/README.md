## 关于开源基础项目的说明

当前项目基于 MIT 许可的 qrcp 开源项目进行二次开发，已加入桌面 GUI、Windows 右键集成、后台 agent、托盘控制、Chat 会话、配置和通知等产品化能力。

### 独立性说明
在申请表的"其他说明"部分，我已经写明：

> 本软件基于 MIT 许可的开源项目 qrcp 进行二次开发，并在合法保留原开源许可与归属声明的基础上形成当前版本。申请登记范围为申请人独立开发、修改和整合形成的当前稳定可演示版本，包括 EQT 桌面 GUI、Windows 右键集成、桌面后台服务、任务队列、历史状态、本地 Chat 会话、附件保存、配置管理、网络接口选择、启动项和系统通知等能力。命令行程序当前仍保留 `eqrcp` 命令名，后续产品化版本计划逐步迁移为 `eqt`。

提交材料不要把 qrcp 上游原始代码表述为完全原创；应把登记范围定位为当前版本中独立开发、修改和整合形成的软件成果。

## 常见问题

### Q1: 基于开源项目二次开发能申请软著吗？
**A**: 可以整理当前版本中独立开发、修改和整合形成的部分申请登记。材料中应如实说明 qrcp 的 MIT 开源来源，保留原许可声明，不把上游原始代码表述为完全原创。

### Q2: 需要提交全部源代码吗？
**A**: 不需要。只需提交前30页+后30页共60页源代码即可。

### Q3: 开源软件能申请软著吗？
**A**: 可以。开源不影响著作权归属，您仍然是著作权人。

### Q4: 申请表中的日期如何填写？
**A**: 
- 开发完成日期：软件基本功能完成的日期
- 首次发表日期：如果未公开发布选"未发表"，如果已发布填写发布日期
- 首次发表日期必须晚于或等于开发完成日期

### Q5: 审查过程中可能遇到什么问题？
**A**: 
- 材料不全：按补正通知补充即可
- 代码格式问题：确保每页50行以上
- 信息不一致：确保所有文档中的信息统一

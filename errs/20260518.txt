  ---
  Q1：STUN 公共 vs 自建

  公共 STUN（推荐起步）

  可用的：
  - stun:stun.l.google.com:19302（默认首选）
  - stun:stun.cloudflare.com:3478
  - stun:global.stun.twilio.com:3478

  优：零成本、零运维、99.9% 可用、全球分布。
  劣：
  - 无 SLA（Google 不承诺，但很少挂）
  - 中国大陆：Google STUN 偶尔被墙、Cloudflare 国内访问不稳定
  - 共享，所有人都在用——不是隐私风险（STUN 只回你的公网地址），但稳定性看运气

  自建 STUN/TURN 要求

  关键区分：
  - STUN = 无状态查询服务："你的公网 IP/端口是 X" + 1 个 UDP 包来回，流量极低
  - TURN = relay 中继服务：P2P 直连失败时，所有数据都过中转服务器，流量贵

  ┌───────────────────────────┬──────────────────────────┬─────────────────────────────────┐
  │                           │           STUN           │              TURN               │
  ├───────────────────────────┼──────────────────────────┼─────────────────────────────────┤
  │ 用户量 1000，每月成本估算 │ $5 VPS 撑得住            │ 取决于流量；100GB 中转 ≈ $10-30 │
  ├───────────────────────────┼──────────────────────────┼─────────────────────────────────┤
  │ 必备资源                  │ 公网 IP（不能在 NAT 后） │ 同左 + 带宽                     │
  ├───────────────────────────┼──────────────────────────┼─────────────────────────────────┤
  │ 端口                      │ UDP 3478                 │ UDP/TCP 3478 + UDP 49152-65535  │
  ├───────────────────────────┼──────────────────────────┼─────────────────────────────────┤
  │ 标准实现                  │ coturn / pion/turn       │ 同左                            │
  └───────────────────────────┴──────────────────────────┴─────────────────────────────────┘

  自建难点：
  1. 必须有公网 IPv4 IP——VPS 都有，但家里 NAT 内的机器不行
  2. 滥用风险：开放的 STUN/TURN 易被用作反射放大攻击或被白嫖；TURN 必须开启认证（time-limited credential, RFC
  7635）
  3. 中国大陆部署：阿里云/腾讯云 VPS 备案（不备案可能被关），月成本 $5-15
  4. HA：单机故障 STUN 直接挂；想冗余得多机房 + DNS 轮询
  5. 监控：连接成功率、ICE gathering 耗时、对称 NAT 比例
  6. 运维更新：coturn 偶有 CVE，需要打补丁

  对 eqrcp/EQT 的实际建议：

  阶段        STUN              TURN
  ─────────────────────────────────────────────────
  Phase 1     公共 Google       不做（连不上就提示用户）
  Phase 2     公共 + 国内自建    暂不做
  Phase 3     多区域自建        coturn @ 1 节点
  Phase 4     商业 SLA          多区域 coturn

  Pro 套餐的护城河实际上在 signaling 服务——STUN 自己用公共的就行，但 signaling endpoint 必须自托管，付费才给
  endpoint URL。这是天然的服务绑定，技术上无法绕过（防破解的核心见 Q5 文档）。
